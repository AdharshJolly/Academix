1.  The Problem

The Scenario

Every B.Tech student drowns in WhatsApp messages trying to remember deadlines shared in group chats. Exam schedules are announced on notice boards, deadlines get buried in 500-message groups, and nobody adds anything to their calendar — because it's too much effort.

Your mission: Build CampusFlow — a smart student hub where AI processes information and n8n automatically sends WhatsApp reminders and creates Google Calendar events, so students never miss a deadline again.

2.  What You Will Build

CampusFlow is a modular web app. Teams build the Core Platform (mandatory) and at least two Feature Modules. Every module must connect to n8n — and every n8n workflow must trigger at least one WhatsApp message or Google Calendar event.

2.1  Core Platform — Mandatory

Student onboarding: name, branch, year, subjects, phone number (for WhatsApp), Google account link.
Central dashboard showing today's schedule, pending tasks, and AI tip of the day.
Task/Deadline CRUD with a field for 'WhatsApp reminder time' and 'Add to Calendar' toggle.
Every task creation triggers a webhook to n8n — which sends a WhatsApp confirmation and creates a Google Calendar event automatically.

4.  n8n Workflow Blueprints

Below are the two core n8n workflows your app must implement. You can use n8n's visual editor to drag-and-drop these nodes — no extra coding required inside n8n.

Workflow 1  —  Deadline Reminder (WhatsApp + Calendar)

➊  Webhook Node  →  Your backend sends: { studentName, phone, subject, deadline, taskTitle }
➋  Set Node  →  Format deadline to ISO 8601, compute reminder time = deadline minus 24h
➌  Google Calendar Node  →  Create event: title=taskTitle, start=deadline, description='CampusFlow reminder'
➍  Wait Node  →  Wait until reminderTime (24 hours before deadline)
➎  Twilio / WhatsApp Node  →  Send: "⏰ Hi {{studentName}}! Your {{subject}} assignment '{{taskTitle}}' is due tomorrow! Check your Google Calendar. — CampusFlow 🎓"
➏  (Bonus) Wait Node x2  →  Send a second WA message 1 hour before deadline as final nudge

Workflow 2  —  Notice Summarizer Broadcast

➊  Webhook Node  →  Receives: { noticeText, eventDate, eventTitle, phoneList[] }
➋  HTTP Request Node (AI Call)  →  POST to OpenAI/Groq: 'Summarize this notice in 3 bullet points: {{noticeText}}'
➌  Set Node  →  Extract AI summary text from response JSON
➍  Google Calendar Node  →  Create event: eventTitle, eventDate, description = AI summary
➎  Loop + WhatsApp Node  →  For each phone in phoneList[], send: '📣 Notice Alert:\n{{aiSummary}}\n\nAdded to your calendar! — CampusFlow'

💡  n8n Quick Setup Guide

Go to n8n.cloud → Sign up free → Click 'New Workflow'
Add a Webhook node → Copy the Test URL → paste it in your backend .env as N8N_WEBHOOK_URL
Add Google Calendar node → Connect via OAuth2 (use a Gmail account) → Set action to 'Create Event'
Add Twilio node → Enter your Twilio Account SID + Auth Token (from twilio.com) → Set WhatsApp sandbox number
Add a Wait node between Calendar and WhatsApp → set to a relative time (e.g., 'date at {{reminderTime}}')
Click 'Execute Workflow' in test mode → from your app, POST to the webhook → verify all nodes turn green
Switch webhook to Production URL → update .env → your full automation is live!

6.  Step-by-Step Build Plan  (3 Hours)

  Phase 1  —  Setup & n8n Configuration
0:00 – 0:30  (30 min)  
Create project repo (frontend/ + backend/ folders). Initialize Node.js + React projects.
Set up .env: AI_API_KEY, SUPABASE_URL, SUPABASE_KEY, N8N_WEBHOOK_URL, TWILIO_SID, TWILIO_TOKEN.
Sign up at n8n.cloud → create two blank workflows named 'Deadline Reminder' and 'Notice Broadcast'.
In n8n Workflow 1: Add Webhook node → copy test URL → paste in .env as N8N_DEADLINE_WEBHOOK.
Connect Google Calendar to n8n (OAuth2) and Twilio WhatsApp Sandbox — test with a dummy POST.

  Phase 2  —  Core Platform
0:30 – 1:00  (30 min)  
Build student onboarding form: name, branch, year, subjects, phone number (WhatsApp), email (for Calendar).
Create /register and /login endpoints. Store student data in Supabase with phone and email fields.
Build the dashboard UI: today's tasks, upcoming deadlines, and an AI tip placeholder.
Implement task CRUD API. Task model must include: title, subject, deadline, reminderTime, addToCalendar (boolean).
On task creation (POST /tasks), immediately POST to N8N_DEADLINE_WEBHOOK — verify n8n receives the data.

  Phase 3  —  AI Integration
1:00 – 1:45  (45 min)  
Build POST /ai/summarize endpoint: takes text → sends to AI API → returns summary JSON.
Build POST /ai/flashcards endpoint: takes notes → returns array of { question, answer } objects.
Connect one AI feature fully to the UI: e.g., paste lecture notes → flashcards appear as flip cards.
Build the Notice Summarizer UI: text area + 'Summarize & Broadcast' button → calls /ai/summarize → shows result.
On 'Broadcast' click, POST to N8N_NOTICE_WEBHOOK with { noticeText, aiSummary, eventDate, phoneList }.

  Phase 4  —  Complete n8n Workflows
1:45 – 2:15  (30 min)  
In n8n Workflow 1 (Deadline): Add Set node → Google Calendar node (create event) → Wait node → Twilio WhatsApp node.
Configure Google Calendar node: title={{taskTitle}}, start={{deadline}}, description='Deadline tracked by CampusFlow'.
Configure Twilio node: to=whatsapp:+{{phone}}, body='⏰ Hi {{studentName}}, your {{subject}} task is due tomorrow! Check your calendar.'
In n8n Workflow 2 (Notice): Add HTTP Request node (call AI) → Set node → Calendar node → Loop → WhatsApp node.
Test BOTH workflows end-to-end: trigger from your UI → verify WhatsApp message received + Calendar event visible.

  Phase 5  —  Polish & Demo Prep
2:15 – 3:00  (45 min)  
Add a second AI module (e.g., Attendance Alerter or Placement Tracker) for bonus scoring.
Add a 'My Automations' page showing recent n8n workflow triggers and their status (success/pending).
Seed 3–5 realistic sample entries (real subject names, real deadlines) so the demo looks live and convincing.
Test the full user journey: register → add deadline → receive WhatsApp → check Google Calendar event created.
Prepare 2-minute pitch: Problem → Solution → Live Demo (show WA on phone) → Tech Architecture → What's next.
